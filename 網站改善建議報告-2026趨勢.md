## 二、2026 趨勢對照與缺口

### 2.1 已具備的優勢

- **響應式版面**：物件列表、篩選、卡片在手機／平板／桌機皆有適配。
- **搜尋與篩選**：行政區、建築類型、房型、關鍵字搜尋；房型按鈕動態產生。
- **多媒體**：物件有照片、Google 地圖、TikTok 影片、貸款試算。
- **連結預覽**：`property-detail` 已補齊 og/twitter meta，並有 Edge Function 依物件注入。
- **模組化起步**：`css/index.css`、`js/index-init.js` 已抽出，有 `模組化方案.md` 可接軌。

### 2.2 與 2026 趨勢的落差

| 趨勢項目           | 2026 常見做法                     | 目前狀態                     |
|--------------------|-----------------------------------|------------------------------|
| 極簡、留白、粗體字 | 少裝飾、大留白、清楚階層          | 漸層、陰影多，inline style 多 |
| 一鍵聯絡 / 黏性 CTA| 底部或側邊固定「來電／LINE」      | 聯絡區在頁尾，需捲到下方     |
| 地圖＋鄰里資訊     | 學區、生活圈、步行分數在同一視圖  | 僅 Google 地圖，無學區／生活圈 |
| Schema 結構化資料  | RealEstateListing、ItemList 等    | 尚未實作                     |
| 語音 / 搜尋友善    | 自然語句 FAQ、在地長尾關鍵字      | 無 FAQ、較少口語化描述       |
| 沉浸式看屋         | 360°、3D、可切換建材              | 靜態照片＋地圖＋TikTok       |
| PWA / 離線         | 可安裝、離線看已存列表            | 無 manifest、無 service worker |
| 無障礙             | 鍵盤、aria、Skip、尊重縮放        | `user-scalable=no`、aria 少  |

## 三、依面向的具體改善建議

### 3.1 設計與視覺（更貼近 2026 極簡、信賴感）

**問題簡述**：inline style 多、漸層與陰影使用頻繁，品牌色未集中管理，不利日後改版。

| 建議 | 說明 | 難度 |
|------|------|------|
| 建立 `css/theme.css` | `:root` 集中 `--color-primary`、`--radius`、`--shadow` 等，逐步取代 magic number | 低 |
| 減少 inline style | 將 `index.html`、`property-detail.html` 中重複的按鈕、卡片、區塊改為 class，搬入 `index.css` / `property-detail.css` | 中 |
| 增加留白、精簡裝飾 | 區塊間 padding、標題與內文間距加大；漸層與陰影收斂，保留重點 CTA 即可 | 低 |
| 字型與階層 | 選定 1～2 個品牌字型（如 Noto Sans TC）、用 `theme.css` 的 `--font-*` 控制標題／內文／小字 | 低 |

**可對應**：`模組化方案.md` Phase 1（`theme.css`、各頁 CSS 抽出）。

---

### 3.2 使用者體驗（更直覺、減少步驟）

**問題簡述**：聯絡與 CTA 偏後段、篩選收合後初次不易發現、缺少「收藏／分享」等立即行動。

| 建議 | 說明 | 難度 |
|------|------|------|
| 黏性底部 CTA 列（手機） | 在 768px 以下顯示固定底條：「來電」「LINE」兩個大按鈕，不擋主要內容 | 中 |
| 一鍵撥號 / LINE 於物件卡 | 卡片或浮框上直接「來電」「LINE」（可依 `tel:`、LINE 連結），減少點進詳情才聯絡 | 低 |
| 篩選區預設與圖示 | 評估「搜尋與篩選」預設展開或半展開，並用圖示（如篩選、地圖）輔助理解 | 低 |
| 麵包屑／上一頁 | `property-detail` 有返回，可加上「首頁 > 物件列表 > 案名」麵包屑，利 SEO 與導航 | 低 |
| 空狀態與載入 | 篩選後 0 筆時明確提示「目前沒有符合的物件，可放寬條件或聯絡我們」；載入中給 skeleton 或統一 loading | 中 |

**可對應**：部分可納入 `js/index-init.js`、`pagination-system.js`、`property-modals.js`。

---

### 3.3 效能與載入

**問題簡述**：多支 JS 同步載入、首屏主要圖片未特別優化、未使用 WebP/AVIF。

| 建議 | 說明 | 難度 |
|------|------|------|
| 首屏物件圖 `fetchpriority="high"` | 第一屏 1～2 張物件主圖加 `fetchpriority="high"`，其餘維持 `loading="lazy"`（你已部分使用 lazy） | 低 |
| 非關鍵 JS 改 `defer` | Supabase、`api-data-loader`、`property-modals`、`pagination-system`、`main-script` 等改為 `defer`，保留必要順序 | 中 |
| 圖片格式 | 後端／Supabase 上傳或轉檔時提供 WebP（或 AVIF），`<img srcset>` 或 `<picture>` 選用；若暫無法改後端，先確保現有圖 `loading`/`decoding` 正確 | 中 |
| 字型與圖示 | Font Awesome、Bootstrap Icons 可改為只載入使用到的 subset，或置換為 SVG sprite，減少 CSS/JS 體積 | 中 |
| CDN | 靜態資源（含 `css/`、`js/`）走 Cloudflare 或既有 CDN，確認快取頭正確 | 低（若已用 Pages 則多半具備） |

**可對應**：`模組化方案.md` 中載入順序、Phase 3 拆 main-script 時一併整理。

---

### 3.4 無障礙（a11y）

**問題簡述**：`user-scalable=no` 影響視力不佳使用者；鍵盤與螢幕閱讀器支援不足；部分互動僅 `onclick`。

| 建議 | 說明 | 難度 |
|------|------|------|
| 移除 `user-scalable=no` | `viewport` 改為 `width=device-width, initial-scale=1.0`，允許縮放 | 低 |
| Skip 連結 | 在 `<body>` 後加「跳到主內容」`<a href="#main">`，`#main` 落在主要列表或 `main` 區塊 | 低 |
| 焦點順序與可見性 | 燈箱、地圖、貸款 modal 開啟時焦點移入、關閉回到觸發元素；`:focus-visible` 明顯外框 | 中 |
| 按鈕與篩選 | 篩選、分頁、收合用 `<button>` 或 `role="button"`＋`tabindex="0"`，必要處補 `aria-expanded`、`aria-label` | 中 |
| 圖片 `alt` | 物件主圖、縮圖使用 `alt="案名 - 照片 1"` 等描述，避免 `alt=""` 或無 `alt` | 低 |

**可對應**：`main-script.js`（燈箱、modal）、`pagination-system.js`（篩選、分頁）、`property-detail.html`、`index.html`。

---

### 3.5 SEO 與可發現性

**問題簡述**：首頁 `og:image` 空、無結構化資料、較少語音／口語化內容。

| 建議 | 說明 | 難度 |
|------|------|------|
| 首頁 og:image / twitter:image | 與 `property-detail` 相同，給預設圖或代表圖 `content="https://.../images/og-home.png"`（建議 1200×630） | 低 |
| Schema 結構化資料 | 首頁：`ItemList` 列出當頁物件（`name`、`url`、`image`）；詳情頁：`RealEstateListing` 或 `Product`（`name`、`description`、`offers`、`image`） | 中 |
| 語音 / 口語友善 | 新增簡短 FAQ（如「楊梅三房多少錢」「楊梅有電梯大樓嗎」），用 `FAQPage` schema；標題與 meta 適度加入口語長尾 | 中 |
| 麵包屑 Schema | 若有麵包屑，可加 `BreadcrumbList` | 低 |

**可對應**：`index.html`、`property-detail.html`、`functions/property-detail.html.js`（若 meta 由 Function 輸出，Schema 也可一併注入）。

---

### 3.6 信任與轉換

**問題簡述**：已有營業員資訊與客戶見證，可再強化「即時性」與「鄰里」以提升信心。

| 建議 | 說明 | 難度 |
|------|------|------|
| 物件更新時間 | 若有 `updated_at` 或類似欄位，在卡片或詳情顯示「X 天內更新」 | 低 |
| 鄰里／生活圈精簡版 | 在詳情頁或浮框加入一兩句（學區、捷運/公車、市場），可先手動或從既有欄位組合，之後再考慮地圖圖層 | 中 |
| 聯絡區與 CTA | 與 3.2 黏性 CTA 一起：確保「立即來電」「LINE」在手機上一指可達 | 低 |

---

### 3.7 行動與 PWA（2026 房產站常見）

**問題簡述**：無 PWA、無離線、無「加到主畫面」；`apple-mobile-web-app-*` 有，但缺乏完整 PWA 基礎。

| 建議 | 說明 | 難度 |
|------|------|------|
| `manifest.json` | `name`、`short_name`、`theme_color`、`background_color`、`icons`（至少 192、512），`display: standalone` 或 `minimal-ui` | 中 |
| Service Worker（可選） |  cache-first 靜態資產、可選離線首頁或「最近看過」列表；可依資源分階段做 | 高 |
| 底部 Thumb Zone CTA | 與 3.2 黏性 CTA 相同：在 768px 以下固定底部，避免被鍵盤或瀏覽器 UI 擋住 | 中 |

---

### 3.8 程式與架構（可維護、可擴充）

**問題簡述**：`admin-dashboard`、`property-admin-db`、`property-detail` 仍內嵌大量 CSS/JS；`main-script`、`pagination-system` 單檔偏大。

| 建議 | 說明 | 難度 |
|------|------|------|
| 依模組化方案推進 | 按 `模組化方案.md` 完成 theme.css、各頁 CSS 抽出，以及 admin / property-detail / property-admin-db 的 JS 抽出 | 中 |
| main-script、pagination 拆分 | Phase 3、4：燈箱、地圖、TikTok、貸款、utils 獨立；分頁可拆 embedded / sold，便於維護與測試 | 中 |
| 事件綁定 | 漸進將 `onclick`、`onfocus`、`onblur` 改為 `addEventListener`，集中在 JS，利於除錯與 a11y | 低～中 |

## 四、建議實施優先順序（矩陣）

依 **效益／成本** 與 **2026 趨勢關聯** 概分：

| 優先級 | 項目 | 理由 |
|--------|------|------|
| P0（立刻可做） | 首頁 `og:image`、`og:url` 補預設值 | 分享首頁時有預覽，實作成本低 |
| P0 | 移除 `user-scalable=no` | 無障礙基本要求，一行改動 |
| P0 | Skip 連結「跳到主內容」 | 鍵盤與讀屏友善，實作快 |
| P1 | 黏性底部 CTA（手機） | 轉換率影響大，符合 2026 行動優先 |
| P1 | 建立 `theme.css` 與 `:root` 變數 | 為後續視覺與模組化打底 |
| P1 | 物件卡片／浮框一鍵來電、LINE | 減少步驟，提升轉換 |
| P1 | 首屏圖片 `fetchpriority="high"`、其餘 `loading="lazy"` 檢視 | 你已有 lazy，補齊 priority 即可 |
| P2 | Schema `ItemList` / `RealEstateListing` | SEO、語音、Rich Result |
| P2 | 非關鍵 JS 改 `defer` | 減少阻塞，提升 LCP |
| P2 | 篩選區 aria、鍵盤、焦點管理 | 與 3.4 無障礙一起 |
| P2 | `manifest.json` 與 PWA 基礎 | 可安裝、品牌露出、部分離線 |
| P3 | 精簡 inline style、增加留白 | 視覺現代化，可與 theme 一起 |
| P3 | 鄰里／學區精簡資訊 | 信任與差異化 |
| P3 | WebP/AVIF、圖示 subset | 效能進階優化 |
| P4 | Service Worker、離線列表 | 資源足夠再規劃 |
| P4 | 360°、3D、AI 推薦 | 中長期，視預算與資料 |

## 六、總結

- 你已在 **響應式、篩選、多媒體、連結預覽（物件頁）、模組化起步** 上做得不錯，和 2026 房產站「行動優先、清楚篩選、多媒體、連結可分享」的基線是對齊的。
- 建議先做 **P0／P1**：首頁 og:image、移除 `user-scalable=no`、Skip、黏性 CTA、`theme.css`、一鍵聯絡。這些改動小、見效快，又能拉高無障礙與轉換。
- 再依資源排 **P2**：Schema、`defer`、aria/鍵盤、`manifest`，強化 SEO、效能與 PWA。
- **P3、P4** 可作為每季迭代：視覺精簡、鄰里資訊、圖片格式、進階 PWA、沉浸式看屋等。

若你需要，我可以把 **P0＋一至二項 P1** 拆成具體 PR/ branch 的 checklist（含檔案與程式碼位置），方便你或團隊逐步實作。
